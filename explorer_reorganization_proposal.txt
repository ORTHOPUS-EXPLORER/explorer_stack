EXPLORER STACK REORGANIZATION PROPOSAL
=====================================
Restructuring to Match Franka ROS2 Architecture Pattern

CURRENT STRUCTURE ANALYSIS
==========================
The current explorer/ package follows a monolithic approach where all functionality
is contained within a single comprehensive package. This differs from the Franka
architecture which uses separate packages for distinct responsibilities.

CRITICAL MIGRATION CONSIDERATIONS
=================================

File Movement Details:
• 8 config files in bringup/config/ need to be distributed across packages
• 4 launch files in bringup/launch/ stay together in explorer_bringup/
• All description files (meshes, URDF, etc.) move as complete directories
• Source code requires careful refactoring of include paths and CMakeLists.txt
• joystick_input.* files move from reference_generator/ to explorer_input_devices/ and rename to xbox_gamepad_joint.*
• Include namespace changes from "ros2_control_explorer" to "explorer_input_devices"

Package Dependencies:
• explorer_bringup depends on ALL other packages (launches the complete system)
• explorer_gazebo depends on explorer_description (needs URDF and meshes)
• Current system continues using orthopus_vesc_interface directly
• explorer_controllers may depend on explorer_msgs if custom messages are used
• explorer_hardware (FUTURE) will coordinate with orthopus_vesc_interface

Build System Changes:
• Each package needs proper CMakeLists.txt with correct find_package() calls
• Header include paths change from "ros2_control_explorer/" to "explorer_*/"
• Shared libraries and executables need updated linking
• Hardware interface development is deferred - no immediate build changes needed

NEW FILES AND THEIR PURPOSE
---------------------------

Package Infrastructure Files (Required for each new package):
• CMakeLists.txt files          - Build system configuration for each package
• package.xml files             - ROS2 package metadata, dependencies, and versioning
• explorer_hardware.xml         - Plugin description file for hardware interface registration
• explorer_gazebo README.md     - Documentation for simulation setup and usage

New Implementation Files:
• explorer_hardware_interface.hpp/cpp  - Hardware abstraction layer that interfaces with
                                         VESC controllers and provides standardized
                                         hardware interface for ROS2 control
• explorer_gazebo.launch.py            - Unified launch file for Gazebo simulation
                                         that combines robot spawning and controller loading

Meta-package Files:
• explorer_ros2 package.xml            - Meta-package for easy installation of entire
                                         Explorer stack with single command

Package Responsibilities:
• explorer_msgs/           - Message, service, action definitions
• explorer_description/    - Robot URDF, meshes, visualization, ROS2 control configuration
• explorer_gazebo/         - Simulation-specific functionality
• explorer_hardware/       - Hardware interface implementation
• explorer_controllers/    - Control algorithms and kinematics
• explorer_input_devices/  - Input device handling (SpaceNav, Xbox gamepad)
• explorer_user_interfaces/ - User interfaces (RQT plugins, web interfaces, mobile apps)
• explorer_bringup/        - System launch and configuration
• explorer_ros2/           - Meta-package for easy installation
